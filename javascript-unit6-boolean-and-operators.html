ΕΝΟΤΗΤΑ 6 JAVASCRIPT - Boolean and Operators

Στο σημερινό δωρεάν μάθημα JavaScript θα δούμε πως ορίζουμε Boolean μεταβλητές και πως χρησιμοποιούμε τους διαφόρους operators που υπάρχουν στην JavaScript όπως logical και comparison.

Στην JavaScript υπάρχουν μόνο δύο τιμές που μπορεί να έχει μια Boolean μεταβλητή – true ή false και χρησιμοποιούνται χωρίς τα double quotes. Εάν κάνετε λάθος και χρησιμοποιήσετε double quotes τότε δεν έχετε μια boolean τιμή αλλά ένα string.

MyScript.js


var a = true;
document.writeln("The type for variable a is: " + typeof a);
document.writeln('</BR');
var b = true;
document.writeln("The type for variable b is: " + typeof b);
document.writeln('</BR');
var c = "true";
document.writeln("The type for variable c is: " + typeof c);

output

	 
The type for variable a is: boolean
The type for variable b is: boolean
The type for variable c is: string

Υπάρχουν τρεις operators (τελεστές) που ανήκουν στην κατηγορία των logical operators και χρησιμοποιούνται σε συνδυασμό με Boolean τιμές (https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Logical_Operators).

Η απλή λογική λέει ότι κάτι που δεν είναι true, τότε θα πρέπει να είναι false. Αν θέλαμε λοιπόν να αντιστρέψουμε την boolean τιμή μιας μεταβλητής, τότε θα πρέπει να χρησιμοποιήσουμε τον Logical NOT ( ! ) operator. Αν τώρα καλέσουμε στον κώδικα μας τον ίδιο logical operator δύο φορές στην ίδια μεταβλητή, τότε θα λάβουμε πίσω σαν απάντηση την αρχική τιμή της μεταβλητής.

MyScript.js


var a = true;
var b = !a;
document.writeln("The type for variable b is: " + b);
document.writeln('</BR>');
var c = !!b;
document.writeln("The type for variable c is: " + c);

output

	 
The type for variable b is: false
The type for variable c is: false

Εάν χρησιμοποιήσουμε έναν logical operator σε μη Boolean τιμή, τότε η τιμή θα μετατραπεί αυτόματα σε Boolean (default συμπεριφορά της JavaScript) για να ολοκληρωθεί η πράξη. Στο πιο κάτω παράδειγμα, η string τιμή “michail” μετατρέπεται αυτόματα σε true και μετά αντιστρέφεται σε false. Αν θέλουμε να πάρουμε true από οποιαδήποτε τιμή απλά χρησιμοποιούμε διπλή άρνηση.

MyScript.js


var a = "michail";
var b = !a;
document.writeln("The type for variable b is: " + b);
document.writeln('</BR>');
var c = "100";
var d = !!c;
document.writeln("The type for variable d is: " + d);

output

	 
The type for variable b is: false
The type for variable d is: true

Όπως μόλις είδαμε, μποορύμε να μετατρέψουμε οποιαδήποτε τιμή σε Boolean αν χρησιμοποιήσουμε διπλή άρνηση. Υπάρχουν όμως έξι συνθήκες που επιστρέφουν false αντί για true και πρέπει να είμαστε ενημερωμένοι για αυτές:

  • empty string “ ”
  • null
  • undefined
  • the number 0
  • the number NaN
  • Boolean false

Ας δούμε όμως μερικά παραδείγματα πως μπορούμε να χρησιμοποιήσουμε τους υπόλοιπους δύο logical operators που είναι ο AND(&&) και ο OR( || ). Ο γενικός κανόνας είναι ότι το AND θα επιστρέψει true όταν όλοι οι όροι του είναι true, ενώ το OR θα επιστρέψει true εφόσον υπάρχει έστω και ένα όρος που είναι true.

MyScript.js


var a1 = true
var a2 = true
var b1 = false
var b2 = false

document.writeln(a1 || b1);
document.writeln('</BR>');
document.writeln(a1 && b1);

output

	 
true
false

Μπορείτε να βρείτε όλους τους συνδυασμούς για το AND και OR στο web site του Mozilla (https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide/Expressions_and_Operators)..

Είναι συχνή προγραμματιστική συνήθεια ο συνδυασμός περισσοτέρων του ενός logical operators.

MyScript.js


var a1 = true
var a2 = true
var b1 = false
var b2 = false
document.writeln(a1 || b1 && a1 && a2);

output

	 
true

Γιατί όμως το παραπάνω παράδειγμα επιστρέφει true? Η απάντηση βρίσκεται στον παρακάτω πίνακα (https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Operator_Precedence) ο οποίος μας διευκρινίζει το γεγονός ότι υπάρχει μια σειρά προτεραιότητας στην εκτέλεση των logical operators.

Η παραπάνω εικόνα μας δείχνει πως το AND προηγείται σε εκτέλεση από το OR οπότε και θα εκτελεστεί πρώτο. Αν τώρα υπάρχουν περισσότερα από ένα && τότε η εκτέλεση αρχίζει από τα αριστερά προς τα δεξιά. Οπότε στο παράδειγμα μας, στην αρχή εκτελείται το b1 && a1 και το αποτέλεσμα αυτής της σύγκρισης θα εκτελεστεί με το a2. Επειδή το b1 είναι false, η εκτέλεση των && επιστρέφει false. Τώρα απομένει το a1 || (αποτέλεσμα από τα &&). Αν και το αποτέλεσμα των && είναι false το OR επιστρέφει true γιατί τουλάχιστον ένας όρος του (το a1) είναι true. Σας ακούγεται λίγο δύσκολο να το ακολουθήσετε? Τότε υπάρχει πιο εύκολος τρόπος: να χρησιμοποιήσετε παρενθέσεις στις πράξεις που θέλετε εσείς να εκτελεστούν πρώτες ορίζοντας έτσι την σειρά εκτέλεσης. Οι παρενθέσεις έχουν υψηλότερη προτεραιότητα εκτέλεσης σε σύγκριση με το AND και OR. Νομίζω με βάση τα όσα έχουμε πει, το παρακάτω παράδειγμα είναι πιο ξεκάθαρο τώρα:

MyScript.js


var a1 = true
var a2 = true
var b1 = false
var b2 = false
document.writeln((a1 || b1) || (a1 && a2));

output

	 
true

Σαν φιλική συμβουλή, θα σας πρότεινα την χρήση των παρενθέσεων (όπου είναι εφικτό) έτσι ώστε να είσαστε απολύτως σίγουροι για το κώδικα σας και το αποτέλεσμα του.

Κάποια στιγμή μπορείτε να ακούσετε για την short-circuit συμπεριφορά της JavaScript όσον αφορά τους logical operators. Τι σημαίνει αυτό? Είναι πάρα πολύ απλό – όταν έχετε πολλαπλούς logical operators στην ίδια πράξη και είναι εμφανές το αποτέλεσμα από την αρχική σύγκριση, τότε η JavaScript δεν θα εκτελέσει ολόκληρη την πράξη έτσι ώστε να μειώσει το execution time. Δηλαδή? Ας πάρουμε το παράδειγμα

true || false || true || false || true

Αφού ο πρώτος όρος είναι true, τότε οποιονδήποτε συνδυασμό OR και να προσπαθήσουμε μετά από αυτό θα πάρουμε true σαν απάντηση. Η JavaScript συγκρίνει τον πρώτο συνδυασμό από τα αριστερά προς τα δεξιά (true || false) και αφού καταλαβαίνει ότι και οι υπόλοιπες συγκρίσεις είναι OR τότε τις παραβλέπει.

Πριν κλείσουμε το σημερινό δωρεάν μάθημα JavaScript, θα ήθελα να αναφερθώ σε ένα ακόμα γκρουπ από operators που ονομάζονται comparison operators. Το αποτέλεσμα τους είναι μια Boolean τιμή αφότου συγκρίνουν τις τιμές (https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide/Expressions_and_Operators).

Αν και αυτοί οι operators θα έχουν μεγάλη αξία για εμάς λίγο αργότερα (όταν μάθουμε τις if/else συγκρίσεις) απλά για τώρα θα ήθελα να επισημάνω μια μεγάλη διαφορά μεταξύ του == και του ===. Στο == operator, πρώτα οι δύο όροι μετατρέπονται αυτόματα από την JavaScript στο ίδιο data type και μετά γίνεται η σύγκριση τους, ενώ το === συγκρίνει άσχετα αν οι δύο όροι έχουν ίδια data types. Αυτή είναι μια μια από τις μικρές λεπτομέρειες που πρέπει να προσέχουμε όταν μελετάμε πίνακες όπως ο παραπάνω.
