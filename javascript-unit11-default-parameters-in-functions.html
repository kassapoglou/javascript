ΕΝΟΤΗΤΑ 11 JAVASCRIPT- DEFAULT PARAMETERS IN FUNCTIONS

Στο σημερινό δωρεάν μάθημα JavaScript θα εξηγήσουμε τι είναι οι default parameters, πως χρησιμοποιούνται, και τι πρέπει να γνωρίζουμε για το εύρος δράσης τους (scope).

Στην JavaScript, οι τιμές των παραμέτρων που ορίζουμε, εάν δεν αναθέσουμε εμείς κάποιες τιμές, θα έχουν σαν default τιμή το undefined. Αυτή η προκαθορισμένη, από την JavaScript, συμπεριφορά συνήθως προκαλεί δυσκολίες στο να βρεθούν λάθη στον κώδικα, για αυτό και σαν εναλλακτική μπορούμε να χρησιμοποιήσουμε default parameters. Δηλαδή να ορίζουμε εμείς τις default τιμές που θα έχουν οι παράμετροι που ορίζουμε σε μια function σε περίπτωση που δεν περάσουμε εμείς κάποια τιμή.

Σαν πρώτο παράδειγμα, στο σημερινό δωρεάν μάθημα JavaScript, θα δούμε τι συμβαίνει όταν δεν ορίσουμε default parameters και καλέσουμε μια function χωρίς να της περάσουμε όλες τις τιμές που περιμένει.

MyScript.js


function multiply(a, b) {
    return a * b
}

var x = multiply(5, 2)  // 10
document.writeln(x);
document.writeln('</BR>');
var y = multiply(5)     // NaN !
document.writeln(y);

Output

		 
10
NaN

Στο παραπάνω παράδειγμα, την δεύτερη φορά που δίνουμε τιμή μόνο για το a, η τιμή του b θα είναι undefined. Πολλαπλασιάζοντας undefined με 5 θα μας επιστρέψει NaN γιατί η JavaScript δεν μπορεί να ολοκληρώσει την πράξη.

Σε παλαιότερες εκδόσεις τις JavaScript, για να λύσουμε το πρόβλημα, εκτελούσαμε ελέγχους στις μεταβλητές για να δούμε αν έχουν τιμή ή είναι undefined. Όμως μετά την υιοθέτηση του ES2015, μπορούμε να ορίσουμε default τιμές στις παραμέτρους της function αποφεύγοντας λάθη στις πράξεις. Για να ορίσουμε default τιμές στις παραμέτρους δεν έχουμε παρά να αναθέσουμε τιμές στις παραμέτρους που βρίσκονται μέσα στην παρένθεση του function. Αν δώσουμε δικές μας τιμές στην function κατά την εκτέλεση του κώδικα μας, τότε οι default τιμές θα αγνοηθούν. Για τις παραμέτρους που δεν δίνουμε εμείς τιμές, θα χρησιμοποιούν οι default τιμές. Ας δούμε τώρα πως αλλάζει ο κώδικας, χρησιμοποιώντας default parameters.

MyScript.js


 function multiply(a = 1, b=1) {
    return a * b
}

var x = multiply(5, 2)  // 10
document.writeln(x);
document.writeln('</BR>');
var y = multiply(5)     // 5
document.writeln(y);
document.writeln('</BR>');
var k = multiply(5, undefined);     // 5
document.writeln(k);

Output

		 
10
5
5

Πρέπει να προσέχουμε λίγο όμως με την χρήση των default parameters και να καταλάβουμε πότε ακριβώς αξιολογούνται. Στην JavaScript, αντίθετα με άλλες γλώσσες προγραμματισμού όπως η Python για παράδειγμα, αξιολογεί τις default parameters μόνο την στιγμή που καλούμε την function στην οποία ανήκουν δημιουργώντας ένα καινούργιο αντικείμενο κάθε φορά. Αυτό σημαίνει ότι η function δεν “θυμάται” προηγούμενες καταστάσεις των μεταβλητών της αφού κάθε κάλεσμα της function είναι ένα καινούργιο αντικείμενο.

MyScript.js


function append(value, array = []) {
    array.push(value)
    return array
}

var x = append(1)  // [1]
document.writeln(x);
document.writeln('</BR>');
var y = append(2)  // [2], not [1, 2]
document.writeln(y);

Output

		 
1
2

Οι default parameters που ορίζονται στην αρχή της παρένθεσης της function είναι διαθέσιμες στις default parameters που ορίζονται στην ίδια παρένθεση αργότερα. Ας δούμε ένα απλό παράδειγμα που εξηγεί καλύτερα αυτή την ιδιότητα τους.

MyScript.js


function greet(name, greeting, message = greeting + ' ' + name) {
    return [name, greeting, message]
}

var x = greet('David', 'Hi');  // ["David", "Hi", "Hi David"]
document.writeln(x);
document.writeln('</BR>');
var y = greet('David', 'Hi', 'Happy Birthday!')  // ["David", "Hi", "Happy Birthday!"]
document.writeln(y);

Output

		 
David,Hi,Hi David
David,Hi,Happy Birthday!

Πριν κλείσουμε το σημερινό δωρεάν μάθημα JavaScript, θα ήθελα να αναφερθώ στο εύρος δράσης των default parameters. Πιο συγκεκριμένα πρέπει να γνωρίζετε το γεγονός ότι οι default parameters έχουν το δικό τους scope που τα όρια του δεν είναι ακριβώς ούτε μέσα στην function αλλά ούτε και έξω από την function. Αν μπορείτε να το φανταστείτε, θεωρητικά βρίσκεται κάπου ανάμεσα. Τι ακριβώς σημαίνει αυτό? Στην ουσία επηρεάζει την τιμή που θέλουμε να δείξουμε σαν τελικό αποτέλεσμα. Ας δούμε ένα παράδειγμα, και θα εξηγήσουμε επάνω σε αυτό τι ακριβώς συμβαίνει.

MyScript.js


var scope="outer_scope";

function scoper(val=scope){
    var scope="inner_scope";
    document.writeln(val); //outer_scope
}
scoper();

Output

		 
outer_scope

Στην παράμετρο val αναθέτουμε μια default τιμή που είναι εκείνη της μεταβλητής scope. Οπότε η val είναι μια default parameter. Μέσα στην function, ορίζουμε τοπικά μια μεταβλητή ξανά με το όνομα scope. Κάτω από άλλες συνθήκες θα περιμέναμε ότι η δεύτερη μεταβλητή θα επισκίαζε την πρώτη, όμως επειδή η default parameter έχει το δικό της scope όπως ήδη αναφέραμε, δεν επηρεάζεται καθόλου από την scope μεταβλητή που δημιουργήθηκε σε inner scope μέσα στην function.
